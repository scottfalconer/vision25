ART DIRECTION / DESIGN PHYSICS

1) Luminance / palette philosophy

- This is a luminous cinematic site, not a dark one.

- The dominant palette should be light-to-mid atmospheric neutrals, with occasional deeper anchor moments for contrast.

- Most sections should feel bright, airy, and sculptural.

- Darker bands can appear as accents or interludes, but must never dominate the experience.

- At least 70% of the overall experience should feel light, luminous, or softly atmospheric rather than dark.

Suggested palette direction:

- mineral white: #F5F7F8

- fog white: #EEF3F6

- pearl silver: #E3EBF0

- pale slate mist: #D7E1E8

- cloud glass: rgba(255,255,255,0.55–0.72)

- deep ink for text: #101820

- soft slate text: #33414D

- cool silver-blue accents: #A8C7DB / #BDD8E7 / #8FB7D1

- occasional deeper anchor bands: #18212A / #1F2B35

Do NOT use:

- flat #000

- dominant #030303

- giant near-black surfaces as the default background

- muddy dark charcoal everywhere

Use drama through:

- light gradients

- luminous bloom

- atmospheric haze

- strong typography

- clean compositional contrast

- bright material reflections

- sculptural spacing

2) Typography tension (expensive contrast)

- Macro headings: massive, tightly controlled, hyper-structured neo-grotesk.

- Prefer:

  - PP Neue Montreal

  - Space Grotesk

  - Clash Display

  - or a very similar modern sans if unavailable

- Micro UI labels + timestamps: crisp utilitarian monospace.

- Prefer:

  - Geist Mono

  - JetBrains Mono

  - or similar if unavailable

Typography should feel:

- architectural

- precise

- expensive

- readable from distance

- modern, not playful

Use readable weights rather than fragile ultra-light ones.

Avoid luxury-thin typography that disappears on a projector.

Large-screen type guidance:

- hero display: roughly 96–160px

- section headings: roughly 48–72px

- body copy: roughly 20–22px

- support copy: roughly 18px

- micro labels / timestamps / pills: roughly 14–16px minimum on large desktop

Headings should feel monumental.

Metadata should feel instrument-panel precise.

Keep line-lengths controlled and paragraphs concise.

3) Materiality

- Use daylight glassmorphism, not dark cyberpunk glass.

- Frosted glass cards should feel luminous, milky, refined, and layered.

- Use backdrop-filter blur around 18px–24px.

- Use brighter translucent surfaces:

  - rgba(255,255,255,0.50+)

  - soft milky whites

  - pearl highlights

- Add elegant edge definition so cards read on projection.

- Borders should be subtle but visible.

- Glass cards must separate clearly from the page background and media.

- Add soft inner highlights and gentle shadows with restraint.

- No chunky shadows.

- No muddy overlays.

Glass direction should feel like:

- frosted acrylic

- museum glass

- polished ice

- architectural glazing

- bright fogged windows with reflected light

4) Grain / texture

- Add extremely subtle film grain/noise only where it helps tactility.

- Keep it around 1–2% opacity.

- Focus grain on media surfaces and interludes, not on text and core UI.

- The result should feel tactile without becoming dirty or soft.

5) Motion choreography

- Motion should feel cinematic, deliberate, slow, and friction-based.

- No bounce.

- No flashy SaaS pop-in.

- Use:

  - masked reveals

  - elegant fades

  - slow vertical drift

  - subtle refractive shifts

  - controlled line motion

  - soft highlight movement on glass

- Hover behavior:

  - cards gently breathe

  - media inside cards scales very subtly

  - light edges shift slightly

- Motion should be visible enough to feel special on a projector, but not noisy.

- Respect prefers-reduced-motion by simplifying or disabling long drift/parallax behaviors.

6) Projection-safe layout principles

- Use a generous safe frame around important content.

- Do not place critical text or controls near the edges.

- Use large gutters and deliberate whitespace.

- On large desktop / projector widths, prefer 2–3 strong columns rather than fussy micro-grid density.

- Bento layouts should feel gallery-like, elegant, and legible.

- Key content should read in bold silhouettes, not in tiny details.

- Use stronger section rhythm and visual pauses.

7) Accessibility / contrast

- Must meet WCAG AA at minimum everywhere.

- Prefer AAA contrast for major text and key UI where practical.

- Never rely on color alone to indicate state.

- All filter controls, toggles, pills, and nav items need obvious active/focus/hover states.

- Text over media must always sit on a controlled contrast surface:

  - scrim

  - glass plate

  - gradient field

  - elevated card

- Use semantic HTML, proper headings, keyboard navigable controls, visible focus styles, and accessible dialogs.
